# Contributing Guide

Thank you for your interest in contributing to the EEase Web API project! There are several ways to contribute to our project, and we welcome all of them.

## How Can You Contribute?

### 1. Issue Reporting

If you find a bug or have a suggestion for improvement:

1. Open a new issue using the GitHub Issue tracker
2. Write a detailed description of the problem or suggestion
3. If possible, include steps to reproduce the issue
4. Add helpful information such as screenshots or error messages

### 2. Code Standards

In our project, we try to adhere to the following standards:

- Clean Code principles
- SOLID principles
- Understandable and maintainable coding
- Unit tests
- Code documentation with XML or /// comment lines

### 3. Development Process

1. First, check existing issues and indicate which one you want to work on
2. Write your code and add tests
3. Review your code before opening a Pull Request
4. When opening a Pull Request, don't forget to explain your changes

## Branch Strategy

- `main`: Only for stable, tested code
- `development`: Active development branch
- `feature/...`: For new features
- `bugfix/...`: For bug fixes
- `release/...`: For release preparations
